# FinSent-CoT: Financial Sentiment Analysis with Chain-of-Thought Reasoning

A high-quality, balanced dataset of **16,944 samples** for training financial sentiment classifiers that explain their reasoning. Each sample pairs a financial text with a structured chain-of-thought (CoT) analysis and a sentiment label (positive, negative, or neutral).

## Key Features

- **Chain-of-Thought Reasoning**: Every sample includes expert-level financial analysis explaining *why* a text is positive, negative, or neutral — averaging 71.6 words of reasoning per sample
- **Perfectly Balanced**: 5,648 samples per class (positive / negative / neutral)
- **Strictly Validated**: All samples pass quality filters for label accuracy, reasoning depth, and financial relevance
- **Multi-Format**: Available in SFT, GRPO, and raw formats for different training pipelines
- **Zero Duplicates**: Deduplicated by text hash across all source datasets
- **Diverse Sources**: Financial tweets, news headlines, earnings reports, and analyst commentary from 4 curated datasets

## Dataset Summary

|-------|---------|----------|----------|---------|

**Total size**: ~52 MB across all formats (raw: 20.7 MB, SFT: 14.8 MB, GRPO: 13.8 MB)

## Generation Pipeline

```
4 Source Datasets → Balanced Sampling → Qwen3-235B-A22B-FP8 (vLLM) → CoT Generation (up to 3 attempts) → Strict Validation → Class Balancing → Final Dataset
```

### Teacher Model

All chain-of-thought reasoning was generated by [**Qwen3-235B-A22B-FP8**](https://huggingface.co/Qwen/Qwen3-235B-A22B-FP8) — Alibaba's 235B parameter Mixture-of-Experts model with 22B active parameters — served via [vLLM](https://github.com/vllm-project/vllm) (v0.8.5) on NVIDIA Hopper H100 GPUs at Indiana University's [Big Red 200](https://kb.iu.edu/d/brcc) supercomputer.

**Generation settings**: `temperature=0.7`, `top_p=0.9`, `max_tokens=512`, with async concurrent requests (16 parallel workers).

The teacher model was prompted to:
1. Analyze financial text for sentiment-relevant indicators (revenue, earnings, market movements, analyst language)
2. Provide structured reasoning in `<reasoning>` tags
3. Classify sentiment as positive, negative, or neutral in `<answer>` tags
4. Use up to 3 attempts per sample with progressively more specific guidance if initial attempts failed quality checks

### Source Datasets

| Source | Dataset | Contribution | Description |
|--------|---------|-------------|-------------|
| Financial Tweets | [TimKoornstra/financial-tweets-sentiment](https://huggingface.co/datasets/TimKoornstra/financial-tweets-sentiment) | 58.7% | Aggregated financial tweets from 9 sub-sources (~38K original) |
| FinGPT | [FinGPT/fingpt-sentiment-train](https://huggingface.co/datasets/FinGPT/fingpt-sentiment-train) | 38.8% | Financial news headlines and tweets (~77K original) |
| Kaggle Financial | [chiapudding/kaggle-financial-sentiment](https://huggingface.co/datasets/chiapudding/kaggle-financial-sentiment) | 1.9% | Mixed financial news and social media (~5.8K original) |
| FiQA | [pauri32/fiqa-2018](https://huggingface.co/datasets/pauri32/fiqa-2018) | 0.6% | Financial opinion mining and QA (~1.1K original) |

All sources were deduplicated by MD5 text hash before generation. Class balancing was applied post-generation — **no oversampling was used**, only unique texts are included.

## Data Formats

The dataset is available in three formats. Use the `config_name` parameter to load a specific format:

```python
from datasets import load_dataset

# SFT format (recommended for supervised fine-tuning)
ds = load_dataset("Ayansk11/FinSent-CoT-Dataset", "sft")

# GRPO format (for reinforcement learning)
ds = load_dataset("Ayansk11/FinSent-CoT-Dataset", "grpo")

# Raw format (all original fields)
ds = load_dataset("Ayansk11/FinSent-CoT-Dataset", "raw")
```

### SFT Format (`sft_*.jsonl`) — For Supervised Fine-Tuning

| Field | Type | Description |
|-------|------|-------------|
| `instruction` | string | System prompt for the financial sentiment analysis task |
| `input` | string | The financial text to analyze |
| `output` | string | CoT reasoning in `<reasoning>` tags + label in `<answer>` tags |

**Example** (real sample from dataset):
```json
{
  "instruction": "You are a financial sentiment analyst. Analyze the given financial text and provide your reasoning in <reasoning> tags and your sentiment classification (positive, negative, or neutral) in <answer> tags.",
  "input": "The offer represents a premium of 24 % over the stock 's closing price on Friday .",
  "output": "<reasoning>\nThe text mentions a \"premium of 24%\" over the stock's closing price, which is a key financial indicator. A premium typically signals positive sentiment as it reflects confidence in the company's value and incentivizes shareholders to accept the offer. The specific percentage (24%) is substantial, suggesting strong buyer interest. The tone is neutral in delivery but implies positivity due to the favorable pricing for investors. Market implications include potential stock price appreciation if the offer is accepted, reinforcing a positive outlook.\n</reasoning>\n<answer>positive</answer>"
}
```

### GRPO Format (`grpo_*.jsonl`) — For Reinforcement Learning

| Field | Type | Description |
|-------|------|-------------|
| `prompt` | string | Full prompt including the financial text and task instruction |
| `answer` | string | CoT reasoning + label (same content as SFT output) |
| `label` | string | Sentiment label: `positive`, `negative`, or `neutral` |

**Example** (real sample from dataset):
```json
{
  "prompt": "Analyze the sentiment of the following financial text:\n\n\"South Africa could fall deeper into junk territory after S&P cut the outlook on its assessment of the nation's debt… https://t.co/ESBc1kS0JG\"\n\nProvide your reasoning in <reasoning> tags and your classification (positive, negative, or neutral) in <answer> tags.",
  "answer": "<reasoning>\nThe text mentions S&P cutting South Africa's debt outlook, a key indicator of deteriorating creditworthiness. The phrase \"fall deeper into junk territory\" uses strongly negative language (\"junk territory\") to signal heightened financial risk. This downgrade implies potential market instability, increased borrowing costs, and reduced investor confidence for South Africa, all of which carry significant negative implications for its economy.\n</reasoning>\n<answer>negative</answer>",
  "label": "negative"
}
```

### Raw Format (`raw_*.jsonl`) — All Original Fields

| Field | Type | Description |
|-------|------|-------------|
| `text` | string | Original financial text from source dataset |
| `label` | string | Ground-truth sentiment label from source |
| `source` | string | Source dataset identifier (`fin_tweets`, `fingpt`, `kaggle_fin`, `fiqa`) |
| `reasoning` | string | Extracted CoT reasoning (without XML tags) |
| `answer` | string | Model's predicted sentiment label |
| `response_raw` | string | Full model response with `<reasoning>` and `<answer>` tags |
| `attempt` | int | Generation attempt number (1, 2, or 3) |
| `issues` | list | Quality issues detected (empty for most samples) |

**Example** (real sample from dataset):
```json
{
  "text": "Congrats to all the $AMD and $GOOG bulls 🍾",
  "label": "positive",
  "source": "fin_tweets",
  "reasoning": "The text congratulates bulls (investors betting on rising prices) for AMD and GOOG stocks, indicating positive outcomes for these investments. The use of \"congrats\" and the champagne emoji 🍾 conveys celebration, suggesting recent gains or successful trades. This implies upward stock price movement for both companies, which is favorable for bulls and signals positive market sentiment toward these tech stocks.",
  "answer": "positive",
  "response_raw": "<reasoning>\nThe text congratulates bulls (investors betting on rising prices) for AMD and GOOG stocks...\n</reasoning>\n<answer>positive</answer>",
  "attempt": 1
}
```

## Quality Assurance

### Validation Pipeline

Every sample in the final dataset passed these strict quality checks:

| Check | Criteria | Pre-Validation Rejection Rate |
|-------|----------|-------------------------------|
| **Label-Answer Match** | Model's predicted answer must exactly match the source ground-truth label | 15.5% (3,964 samples rejected) |
| **Non-Empty Reasoning** | Must contain substantive CoT analysis (not empty or placeholder) | 8.6% (2,208 samples rejected) |
| **Minimum Reasoning Length** | At least 20 words of financial analysis | < 1% |
| **Financial Relevance** | Must contain financial or analytical terminology | < 1% |
| **No Input Copying** | Reasoning must not simply paraphrase the input text (< 80% word overlap) | < 1% |
| **Analysis over Filler** | Financial analysis terms must outweigh generic filler phrases | < 1% |

### Validated Dataset Statistics

## Intended Use

This dataset is designed for fine-tuning small language models (0.5B–8B parameters) to perform financial sentiment analysis with interpretable, step-by-step reasoning. The target deployment is on-device inference via [Ollama](https://ollama.ai/) or [llama.cpp](https://github.com/ggerganov/llama.cpp) on consumer hardware (e.g., Apple Silicon Macs).

### Training Pipeline

```
FinSent-CoT-Dataset
    │
    ├── Phase 1: SFT (Supervised Fine-Tuning)
    │   └── Teaches the model the CoT output format and financial reasoning patterns
    │       using instruction/input/output pairs
    │
    └── Phase 2: GRPO (Group Relative Policy Optimization)
        └── Refines reasoning quality using 4 equal-weight reward functions:
            ├── Correctness — predicted label matches ground truth
            ├── Format — proper <reasoning>/<answer> XML tag structure
            ├── Reasoning Quality — depth, financial terminology, analytical rigor
            └── Consistency — reasoning logically supports the final answer
```

### Target Models

| Model | Parameters | Training Backend | Deployment |
|-------|-----------|-----------------|------------|
| [Qwen3-0.6B](https://huggingface.co/Qwen/Qwen3-0.6B) | 0.6B | Unsloth QLoRA | Ultra-fast inference |
| [MobileLLM-R1-950M](https://huggingface.co/facebook/MobileLLM-R1-950M) | 0.95B | PEFT + bitsandbytes | Edge/mobile devices |
| [DeepSeek-R1-Distill-Qwen-1.5B](https://huggingface.co/deepseek-ai/DeepSeek-R1-Distill-Qwen-1.5B) | 1.5B | Unsloth QLoRA | R1 reasoning capability |
| [Qwen3-1.7B](https://huggingface.co/Qwen/Qwen3-1.7B) | 1.7B | Unsloth QLoRA | Good speed/quality balance |
| [Qwen3-4B](https://huggingface.co/Qwen/Qwen3-4B) | 4B | Unsloth QLoRA | Recommended default |
| [Qwen3-8B](https://huggingface.co/Qwen/Qwen3-8B) | 8B | Unsloth QLoRA | Highest quality |

All models are exported to GGUF format in three quantizations: **Q4_K_M**, **Q5_K_M** (recommended), and **Q8_0**.

## Limitations

- **English only**: All texts and reasoning are in English
- **3-class granularity**: Sentiment is classified as positive/negative/neutral only — no intensity levels (e.g., "strongly positive")
- **Source distribution**: 58.7% of data originates from financial tweets, which tend to be informal, short, and use stock ticker symbols ($AMD, $TSLA)
- **Teacher model ceiling**: Reasoning quality is bounded by Qwen3-235B-A22B's capabilities and biases
- **Temporal scope**: Source datasets span various time periods (2014–2025); no explicit temporal filtering was applied
- **Domain scope**: Focused on equities, crypto, and corporate earnings; may not generalize to insurance, real estate, or other financial sub-domains
- **No multi-label support**: Each text receives exactly one sentiment label
